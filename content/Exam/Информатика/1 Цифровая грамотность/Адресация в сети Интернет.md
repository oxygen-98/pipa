Адресация в сети Интернет — это процесс присвоения уникальных идентификаторов устройствам, подключенным к сети, для обеспечения возможности их нахождения и связи друг с другом. Основным типом адресации в Интернете является IP-адресация, которая реализуется с помощью двух версий протокола IP: IPv4 и IPv6.

### IPv4

IPv4 — это первая версия протокола IP, которая была разработана в 1980-х годах. Адрес IPv4 состоит из 32 бит, представляемых в виде четырех октетов, разделенных точками. Например, 192.168.1.1. Каждый октет может принимать значения от 0 до 255.

Однако из-за ограниченности количества возможных адресов (около 4 миллиардов) и быстрого роста числа устройств, подключающихся к Интернету, возникла необходимость перехода на новую версию протокола.

### IPv6

IPv6 — это новая версия протокола IP, разработанная для решения проблемы нехватки адресов. Адрес IPv6 состоит из 128 бит, представленных в виде восьми групп по четыре шестнадцатеричных цифры, разделенных двоеточиями. Например, fe80::1.

### DNS (Система доменных имен)

Для удобства использования числовых IP-адресов была создана система доменных имен (DNS). DNS позволяет пользователям использовать понятные имена, такие как example.com, вместо сложных числовых адресов. Когда пользователь вводит имя сайта в браузере, DNS-сервер преобразует это имя в соответствующий IP-адрес, позволяя устройству установить соединение с нужным сервером.

### DHCP (Dynamic Host Configuration Protocol)

DHCP — это протокол, используемый для автоматической настройки сетевых параметров устройства, включая назначение IP-адреса. Это упрощает администрирование сетей, так как администраторам не нужно вручную настраивать каждое устройство.

### NAT (Network Address Translation)

NAT используется для преобразования частных IP-адресов внутри локальной сети в публичные IP-адреса, видимые в Интернете. Это позволяет нескольким устройствам использовать один публичный IP-адрес, что помогает экономить ограниченное пространство адресов IPv4.
