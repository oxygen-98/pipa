Система доменных имён (DNS, Domain Name System) — это распределённая иерархическая система для сопоставления имен хостов (доменов) с их IP-адресами. DNS играет ключевую роль в работе интернета, позволяя пользователям обращаться к сайтам и службам по удобным для восприятия именам, вместо использования сложных числовых IP-адресов.

### Принцип работы DNS

1. **Запрос**: Пользователь вводит доменное имя (например, `example.com`) в браузер или другое приложение.
2. **Обращение к DNS-серверу**: Браузер обращается к ближайшему DNS-серверу (чаще всего предоставляемому провайдером), чтобы узнать IP-адрес соответствующего домена.
3. **Поиск IP-адреса**: DNS-сервер проверяет свою кэшированную базу данных. Если записи нет, он направляет запрос к вышестоящим DNS-серверам, пока не найдет нужную информацию.
4. **Ответ**: После получения IP-адреса DNS-сервер возвращает его браузеру.
5. **Доступ к сайту**: Браузер использует полученный IP-адрес для установления соединения с сервером и загрузки запрашиваемого ресурса.

### Иерархия DNS

DNS организована в виде дерева, где корневые серверы находятся на вершине, а дочерние зоны распределяют ответственность за конкретные домены.

1. **Корневые серверы**: Управляют доменными зонами верхнего уровня (.com, .org, .ru и т.д.).
2. **Авторитетные серверы**: Ответственны за конкретные домены и поддомены.
3. **Рекурсивные серверы**: Выполняют запросы от клиентов, последовательно обращаясь к вышестоящим серверам.

### Доменные зоны

1. **Верхнего уровня (TLD)**: .com, .net, .org, .ru и т.д.
2. **Второго уровня**: example.com, google.ru и т.д.
3. **Поддомены**: subdomain.example.com, mail.google.com и т.д.

### Типы записей DNS

1. **A-запись**: Связывает доменное имя с IPv4-адресом.
2. **AAAA-запись**: Связывает доменное имя с IPv6-адресом.
3. **MX-запись**: Указывает на серверы для обработки электронной почты.
4. **CNAME-запись**: Создает псевдоним для другого доменного имени.
5. **TXT-запись**: Содержит произвольную текстовую информацию, например, для подтверждения владения доменом.

### Безопасность DNS

1. **DNSSEC (DNS Security Extensions)**: Технология, обеспечивающая аутентичность и целостность данных DNS с помощью цифровых подписей.
2. **DNS over HTTPS (DoH)**: Передача запросов DNS через защищённый HTTPS-канал для повышения конфиденциальности и безопасности.
